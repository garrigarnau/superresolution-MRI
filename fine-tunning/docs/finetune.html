Fine-Tuning Swin2SR

Adaptació del model pre-entrenat Swin2SR (x4) al domini d'imatges de ressonància magnètica cerebral mitjançant fine-tuning complet amb pèrdua L1 + perceptual

1. Objectiu

El benchmark zero-shot va demostrar que Swin2SR, entrenat amb imatges naturals RGB, es pot aplicar a MRI però no captura les característiques específiques del teixit cerebral. El fine-tuning adapta els pesos del model al nostre domini per millorar PSNR i SSIM.

  • Model base: caidas/swin2SR-classical-sr-x4-64 (HuggingFace)
  • Estratègia: Fine-tuning complet (tots els pesos descongelats)
  • Adaptació: Modificació de les capes d'entrada/sortida per treballar amb 1 canal (grayscale)

2. Pipeline General

Swin2SR Pre-entrenat 3ch RGB, imatges naturals
→
Adaptació 1ch Modificar convs entrada/sortida
→
Fine-Tuning L1 + Perceptual, CUDA
→
Best Checkpoint Max Val PSNR

3. Adaptació del Model a Grayscale

Problema

Swin2SR espera imatges RGB (3 canals). Les nostres MRI són grayscale (1 canal). Convertir a RGB i tornar és ineficient i afegeix soroll.

Solució: Modificar les capes convolucionals

Modifiquem directament l'arquitectura per operar amb 1 canal, preservant al màxim els pesos pre-entrenats:

Primera convolució (entrada)
model.swin2sr.first_convolution: Conv2d(3, 180) → Conv2d(1, 180)
Inicialització: Mitjana dels pesos dels 3 canals d'entrada → 1 canal.
Última convolució (sortida)
model.upsample.final_convolution: Conv2d(64, 3) → Conv2d(64, 1)
Inicialització: Mitjana dels pesos dels 3 canals de sortida → 1 canal.
Tensor de mitjana
model.swin2sr.mean: Shape (1, 3, 1, 1) → (1, 1, 1, 1)
Mitjana dels valors RGB originals.
Per què és segur? El backbone Transformer opera sobre la dimensió d'embedding (180), no sobre els canals de la imatge. Només les capes extremes (primera i última conv) toquen els canals, i la resta de l'arquitectura queda intacta.

4. Funció de Pèrdua

L1 + VGG Perceptual Loss

L1 Loss Fidelitat píxel a píxel
+
Perceptual Loss Qualitat visual (VGG19)
=
Loss Total 1.0·L1 + 0.1·Perceptual

Perceptual Loss (VGG19)

S'extreuen features de 3 capes de VGG19 pre-entrenat (ImageNet):

Capa VGG19ÍndexCaptura
relu1_2[:3]Textures fines, vores
relu2_2[3:8]Patrons locals
relu3_4[8:17]Estructures de nivell mitjà

Com que VGG19 espera 3 canals, les imatges grayscale es repliquen a RGB (.repeat(1, 3, 1, 1)) abans de passar per VGG. Els pesos de VGG es congelen (no s'entrenen).

6. Flux d'Entrenament

Per cada època:

Batch LR (B, 1, 64, 64)
→
Swin2SR Forward pass
→
SR Output (B, 1, 256, 256)
→
Loss vs HR L1 + Perceptual
→
Backprop AMP + GradScaler

Validació (cada època):

  • Inferència sobre tot el validation set (557 imatges) sense gradients
  • Càlcul de PSNR mitjà (dB) com a mètrica principal
  • Si PSNR millora → guardar checkpoint (best_model.pth)
  • Si no millora en 5 èpoques → reduir learning rate ×0.5
  • Si no millora en 15 èpoques → early stopping

10. Decisions de Disseny

Per què fine-tuning complet i no parcial?

El domini MRI és molt diferent de les imatges naturals (ImageNet). Les features apreses per les capes profundes del Transformer (textures naturals, colors, objectes) no són directament transferibles. Un fine-tuning complet permet que tot el model s'adapti a les característiques del teixit cerebral.

Per què learning rate tan baix (2e-5)?

Tot i que desfem tots els pesos, volem partir d'una bona inicialització i no destruir-la. Un LR baix amb AdamW permet una adaptació gradual sense oblidar les capacitats de super-resolució apreses.

Per què L1 + Perceptual i no només L1?

L1 sol produeix resultats nítids a nivell de mètrica (PSNR) però pot generar imatges una mica borroses perceptualment. La pèrdua perceptual (VGG) afegeix pressió per mantenir textures i estructures fines del cervell, millorant la qualitat visual sense sacrificar massa PSNR.

Per què modificar l'arquitectura (1ch) i no duplicar canals?

Duplicar grayscale a RGB per l'entrada i fer la mitjana a la sortida funciona, però: (1) triplica la memòria de les activacions inicials, (2) el model ha d'aprendre que els 3 canals són idèntics, (3) és conceptualment més net operar directament en 1 canal. La inicialització per mitjana dels pesos RGB preserva la informació pre-entrenada.

Per què no data augmentation?

Les MRI ja estan registrades a l'espai Talairach (orientació i escala estandarditzades). Augmentations com flips o rotacions produirien anatomies no realistes. El dataset (4,461 imatges) és suficient per al fine-tuning d'un model ja pre-entrenat.
