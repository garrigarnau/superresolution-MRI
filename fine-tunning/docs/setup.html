Guia d'Execució

Passos per executar el fine-tuning de Swin2SR en una màquina amb GPU CUDA

Requisits previs

  • GPU CUDA amb almenys 8 GB de VRAM
  • Python 3.10+
  • Dades preprocessades a data/train/, data/val/ (generades amb scripts/preprocess.py)
  • Connexió a internet (per descarregar el model Swin2SR i els pesos VGG19 la primera vegada)

Pas 1 — Instal·lar dependències

Des del directori arrel del projecte:

# Crear i activar l'entorn virtual
python -m venv venv
source venv/bin/activate

# Instal·lar totes les dependències
pip install -r requirements.txt

Paquets clau per al fine-tuning:

PaquetVersió mínimaÚs
torch≥ 2.0Backend d'entrenament + CUDA
torchvision≥ 0.15VGG19 per la perceptual loss
transformers≥ 4.30Model Swin2SR
scikit-image≥ 0.21Càlcul de PSNR
Important: Assegura't que PyTorch està instal·lat amb suport CUDA. Comprova amb: python -c "import torch; print(torch.cuda.is_available())". Si retorna False, instal·la la versió CUDA: pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

Pas 2 — Test de validació (GPU stress test)

Abans d'entrenar tota la nit, executa el test de pipeline per verificar que tot funciona correctament amb la teva GPU:

cd fine-tunning
python test_pipeline.py --gpu

Què verifica el test?

TestQuè comprova
[1/7] DatasetLes imatges carreguen, shapes correctes (LR: 1×64×64, HR: 1×256×256)
[2/7] ModelSwin2SR carrega i l'adaptació a 1 canal és correcta
[3/7] ForwardEl model produeix una sortida amb les dimensions esperades
[4/7] LossL1 + Perceptual (VGG19) es calcula sense errors
[5/7] BackwardGradients es computen i l'optimizer fa un step
[6/7] CheckpointGuardar i carregar pesos funciona
[7/7] GPU Stress20 steps reals amb AMP, batch size real, validació completa

Què buscar a la sortida:

  • ALL TESTS PASSED → Tot correcte, pots entrenar
  • Loss is NaN or Inf → Problema amb AMP. Prova desactivar-lo a config.py (USE_AMP = False)
  • CUDA out of memory → Redueix BATCH_SIZE a config.py (prova 4 o 2)
  • Loss may be diverging → Redueix LEARNING_RATE a config.py

Informació útil que et dóna:

  • Peak GPU memory (per saber si hi ha marge)
  • Temps per step i per epoch estimat
  • Temps total estimat per 100 epochs
Opcional: Per un test més llarg, usa --steps 50 per fer 50 steps en lloc de 20.

Pas 3 — Llançar l'entrenament

Un cop el test passa, llança l'entrenament:

cd fine-tunning
python train.py

Per executar en background (si et desconnectes de la sessió SSH):

# Amb nohup (output a nohup.out)
nohup python train.py > training.log 2>&1 &

# O amb screen/tmux
screen -S training
python train.py
# Ctrl+A, D per desconnectar

Pas 4 — Verificar resultats

Un cop acabat l'entrenament, trobaràs:

fine-tunning/checkpoints/
├── best_model.pth   ← Millor model (usar aquest!)
└── last_model.pth   ← Últim epoch (backup)

El checkpoint conté:

  • epoch — En quina època es va guardar
  • model_state_dict — Pesos del model
  • optimizer_state_dict — Estat de l'optimizer (per reprendre)
  • best_psnr — Millor PSNR aconseguit en validació

Resum ràpid

# 1. Instal·lar
pip install -r requirements.txt

# 2. Testar (a la màquina amb GPU)
cd fine-tunning
python test_pipeline.py --gpu

python train.py

# (Opcional) En background:
nohup python train.py > training.log 2>&1 &

Troubleshooting

ProblemaSolució
CUDA out of memoryReduir BATCH_SIZE a config.py (8 → 4 → 2)
Loss NaN/InfDesactivar AMP: USE_AMP = False a config.py
Loss no baixaAugmentar LEARNING_RATE (2e-5 → 1e-4)
Loss explotaReduir LEARNING_RATE (2e-5 → 5e-6)
torch.cuda.is_available() = FalseReinstal·lar PyTorch amb CUDA: pip install torch --index-url https://download.pytorch.org/whl/cu121
Dataset buitExecutar python scripts/preprocess.py primer
Connexió cau durant la nitUsar nohup, screen o tmux
