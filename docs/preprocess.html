Preprocessament MRI

Script de preparació de dades per al projecte de Super-Resolució amb imatges de ressonància magnètica (OASIS dataset)

1. Objectiu

L'script scripts/preprocess.py converteix volums 3D de ressonància magnètica (format Analyze .hdr/.img) en parelles d'imatges 2D PNG per entrenar un model de super-resolució:

  • → HR (High Resolution): Imatges 256×256 px — Ground Truth
  • → LR (Low Resolution): Imatges 64×64 px — Entrada del model (4× reducció bicúbica)

2. Pipeline

Volum 3D .hdr/.img (176×208×176)
→
Extracció Slices Axial (eix Z)
→
Filtratge >1% píxels cervell
→
Normalització 0-255 uint8
→
HR (256²) Resize bicúbic
→
LR (64²) Downsample 4×

3. Dades d'entrada

Dataset OASIS (disc1/)

Utilitzem els volums processats en espai Talairach amb màscara cranial i correcció de biaix:

disc1/OAS1_XXXX_MR1/PROCESSED/MPRAGE/T88_111/*_masked_gfc.hdr/.img

4. Funcions de l'script

load_volume(hdr_path)
Carrega un volum Analyze amb nibabel i elimina dimensions extres (squeeze).
normalize_slice(slice_2d)
Normalitza min-max un tall 2D a rang [0, 255] com a uint8.
is_valid_slice(slice_2d, threshold)
Comprova que el tall tingui més d'un 1% de píxels no-zero (descarta talls buits o amb poc cervell).
extract_slices(volume)
Itera per l'eix axial del volum i retorna tots els talls vàlids normalitzats.
create_hr_lr_pair(slice_2d)
Redimensiona el tall a 256×256 (HR) i després el redueix a 64×64 (LR) amb interpolació bicúbica.
main()
Orquestra tot el procés: descobreix subjectes, extreu talls, genera parelles HR/LR, barreja, divideix i guarda.

9. Decisions de disseny

Per què volums T88_masked_gfc?

Aquests volums ja estan: (1) registrats a un espai comú (Talairach), el que garanteix consistència entre subjectes; (2) amb el crani eliminat, de manera que el model se centra en teixit cerebral; (3) amb correcció de camp de biaix (inhomogeneïtat del camp magnètic).

Per què talls axials?

Els talls axials (transversals) mostren la millor secció del cervell amb ambdós hemisferis visibles, i són els més usats en neuroimatge clínica.

Per què normalització per tall?

Cada tall es normalitza individualment a [0, 255] per maximitzar el contrast local i evitar que talls amb menys senyal quedin foscos.

Per què barrejar abans de dividir?

Barregem tots els talls (de tots els subjectes) i després dividim. Això assegura que cada split conté talls de múltiples subjectes, millorant la generalització del model.
