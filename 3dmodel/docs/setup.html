Super-Resolució 3D

Guia completa per executar el pipeline de super-resolució 3D amb MedicalNet ResNet3D-50 sobre el dataset OASIS

Visió General

A diferència del pipeline 2D (que talla el volum en slices independents), aquest experiment treballa directament amb volums 3D, preservant la coherència inter-slice.

Volum HR 176×208×176
→
Degradació Blur + Downsample ×2
→
Volum LR 88×104×88
→
Model 3D SR ResNet3D + Decoder
→
Volum SR 176×208×176

Arquitectura del model

LR Patch (1, 1, 32³)
→
MedicalNet Encoder ResNet3D-50 (congelat)
→
Multi-scale Features 64→256→512→1024→2048 ch
→
3D Decoder Skip connections + UpConv
→
HR Patch (1, 1, 64³)

Requisits

  • GPU CUDA amb almenys 8 GB VRAM
  • Python 3.10+
  • Dataset OASIS a disc1/
  • Connexió a internet (primer cop: descarrega MedicalNet ~180 MB)

Pas 2 — Preprocessament

cd 3dmodel
python preprocess.py

Què fa:

  • Carrega volums 3D OASIS (176×208×176, normalitzats a [0,1])
  • Genera volums LR: Gaussian blur (σ=1.0) + downsample ×2 → 88×104×88
  • Extreu parelles de patches 3D: HR (64³) i LR (32³)
  • Filtra patches amb poc cervell (<1% vòxels no-zero)
  • Divideix per subjecte: 75% train / 10% val / 15% test
  • Guarda patches com .npy + volums de test complets

Sortida esperada:

3dmodel/patches/
├── train/ HR/*.npy, LR/*.npy
├── val/   HR/*.npy, LR/*.npy
├── test/  HR/*.npy, LR/*.npy
└── test_volumes/ *_hr.npy, *_lr.npy

Pas 3 — Test de validació (GPU)

python test_pipeline.py --gpu

Què verifica:

TestComprova
[1/5] ModelMedicalNet carrega, encoder congelat, decoder trainable
[2/5] ForwardInput 32³ → Output 64³ (×2 upscale correcte)
[3/5] BackwardGradients flueixen només pel decoder
[4/5] DatasetPatches carreguen amb shapes correctes
[5/5] CheckpointSave/load funciona
[GPU] Stress10 steps amb batch real, memòria GPU, NaN check
Si falla OOM: Reduir BATCH_SIZE a config.py (4 → 2 → 1)

Pas 4 — Entrenar el decoder

# Entrenar (foreground)
python train.py

# O en background per sessió SSH
nohup python train.py > training.log 2>&1 &

Configuració:

ParàmetreValor
EPOCHS50 (amb early stopping a 12)
BATCH_SIZE4
LEARNING_RATE1e-4
LOSSL1
OPTIMIZERAdamW
ENCODERCongelat (0 params trainables)
DECODER~15M params trainables

L'encoder (MedicalNet ResNet3D-50, 46M params) es manté congelat. Només s'entrena el decoder 3D (~15M params), el que fa l'entrenament molt més ràpid i viable amb poques dades.

Pas 5 — Inferència amb sliding window

python infer.py

Què fa:

  • Carrega els volums LR de test complets (88×104×88)
  • Model SR: Sliding window amb patches 32³, overlap 8 → reconstrueix volum SR (176×208×176)
  • Trilinear baseline: Simple interpolació trilineal del volum LR → mateixa mida HR
  • Guarda ambdós resultats com .npy

Sortida:

3dmodel/results/
├── model_sr/     ← Volums reconstruïts pel model
└── trilinear/    ← Baseline (interpolació trilineal)

Pas 6 — Avaluació

python evaluate.py

Mètriques calculades:

PSNR 3D (dB)

Fidelitat vòxel a vòxel sobre tot el volum 3D.

SSIM 3D

Similitud estructural 3D (preservació de formes anatòmiques).

Comparació:

MètodeDescripció
Model SRMedicalNet encoder + decoder entrenat amb patches 3D
TrilinearInterpolació trilineal simple (baseline sense IA)

Resultats guardats a results/metrics_3d.json amb mètriques per subjecte i mitjanes globals.

Troubleshooting

ProblemaSolució
CUDA OOMReduir BATCH_SIZE a config.py (4→2→1)
Pocs patches generatsReduir PATCH_STRIDE (48→32) per més overlap
Model no descarregaVerificar connexió internet, trust_remote_code=True
Inferència lentaAugmentar INFER_PATCH_OVERLAP (menys patches però menys qualitat en fronteres)
PSNR molt baixEntrenar més èpoques, reduir LR, augmentar patches (més subjectes)
