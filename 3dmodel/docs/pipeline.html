Super-Resolució 3D MRI

Reconstrucció volumètrica de ressonàncies magnètiques cerebrals utilitzant MedicalNet ResNet3D-50 com a extractor de característiques 3D i un decoder entrenat amb patches volumètrics

1. Motivació

Per què 3D?

En el pipeline 2D anterior (Swin2SR, Real-ESRGAN), cada tall axial es processa de forma independent. Això ignora la relació espacial entre talls consecutius — el cervell és una estructura 3D contínua, i els talls adjacents comparteixen informació anatòmica.

Un model 3D pot aprofitar el context volumètric: les relacions entre vòxels veïns en totes 3 dimensions, millorant potencialment la reconstrucció de detalls anatòmics fins.

Objectiu de l'experiment

  • Entrenar un model de super-resolució 3D nàtiva (×2) sobre volums MRI cerebrals
  • Comparar el rendiment contra una interpolació trilineal (baseline sense IA)
  • Avaluar amb mètriques 3D (PSNR volumètric, SSIM volumètric)

4. Fase 1 — Preprocessament

Generació del volum de baixa resolució (LR)

Per simular una degradació realista, apliquem un pipeline de degradació 3D al volum original (HR):

Volum HR 176×208×176
→
Gaussian Blur 3D σ = 1.0
→
Downsample ×2 Trilineal
→
Volum LR 88×104×88

El blur gaussià 3D simula la pèrdua de freqüències altes que passa en adquisicions de baixa resolució reals. El downsample trilineal redueix la mida per un factor de 2 en cada dimensió.

Extracció de patches 3D

Els volums complets no caben a la memòria de la GPU (un sol volum a float32 ≈ 25 MB, però les activacions intermèdies del model necessitarien >48 GB). Per tant, treballem amb patches — cubs 3D petits extrets del volum:

Patch HR 64 × 64 × 64 vòxels
↔
Patch LR 32 × 32 × 32 vòxels

Un "patch" és simplement un petit cub tallat del volum gran. Com tallar un meló en daus petits. Cada parell LR/HR correspon a la mateixa regió anatòmica del cervell.

ParàmetreValorEfecte
HR patch size64³Prou gran per capturar estructura local
LR patch size32³Correspon a la mateixa regió a meitat de resolució
Stride48Overlap de 16 vòxels entre patches veïns
Brain threshold1%Descarta patches buits (fora del cervell)

Amb 29 subjectes d'entrenament i stride 48: ~800 patches d'entrenament. Reduint el stride a 32 s'obtindrien ~2.500+ patches.

Split per subjecte

La divisió es fa a nivell de subjecte (no de patch) per evitar data leakage — tots els patches d'un cervell van al mateix split:

5. Fase 2 — Model i Entrenament

Arquitectura: Encoder-Decoder amb Skip Connections

El model combina un encoder pre-entrenat (MedicalNet ResNet3D-50) amb un decoder 3D entrenable, connectats amb skip connections similars a una U-Net:

LR Patch (1, 32³)
→
Encoder ResNet3D-50 (congelat)
→
Features Multi-escala
→
Decoder 3D UpConv (entrenable)
→
HR Patch (1, 64³)

Encoder: MedicalNet ResNet3D-50

Pre-entrenat Congelat 46M params HuggingFace

Model pre-entrenat en imatges mèdiques 3D (CT/MRI). Extreu característiques geomètriques i morfològiques a múltiples escales. Els seus pesos NO s'actualitzen durant l'entrenament.

Decoder: 3D Upsampling Path

Cadena de blocs ConvTranspose3d que reconstrueixen progressivament la resolució espacial, combinant amb features de l'encoder via skip connections (com una U-Net 3D):

Per què congelar l'encoder? Amb ~800 patches d'entrenament, entrenar 46M params d'encoder causaria overfitting sever. Congelant-lo, només entrenem el decoder (~15M params), que és suficient per aprendre el mapping features→vòxels. L'encoder ja sap extreure característiques anatòmiques 3D gràcies al seu pre-entrenament en imatges mèdiques.

6. Fase 3 — Inferència (Sliding Window)

Durant l'entrenament treballem amb patches individuals. Però per avaluar, necessitem reconstruir el volum complet. Ho fem amb una finestra lliscant 3D:

Volum LR complet 88×104×88
→
Sliding Window Patches 32³, overlap 8
→
SR per patch 32³ → 64³
→
Stitching Mitjana zones overlap
→
Volum SR 176×208×176

Com funciona la finestra lliscant:

  1. Recorre tot el volum LR extraient patches de 32³ amb stride 24 (overlap 8)
  2. Cada patch passa pel model → produeix un patch SR de 64³
  3. Col·loca cada patch SR a la posició corresponent del volum de sortida (×2 coordenades)
  4. Les zones on es solapen múltiples patches es promitgen → transicions suaus

Baseline: Interpolació Trilineal

Com a control, també generem una reconstrucció simplement ampliant el volum LR amb interpolació trilineal (zoom ×2 matemàtic, sense cap model). Això permet mesurar quant aporta la IA respecte a un mètode clàssic.

7. Fase 4 — Avaluació

Mètriques 3D

Les mètriques es calculen sobre el volum 3D complet, no per talls individuals:

PSNR 3D (dB)

Peak Signal-to-Noise Ratio calculat sobre tots els vòxels del volum. Mesura la fidelitat numèrica global de la reconstrucció.

SSIM 3D

Structural Similarity sobre el volum 3D. Mesura la preservació d'estructures anatòmiques (sulcus, ventricles, substància blanca/grisa).

10. Decisions de Disseny

Per què ×2 i no ×4?

En 3D, un factor ×4 implica 4³ = 64× més vòxels a reconstruir. Amb ×2 (8× més vòxels) el problema és molt més tractable: l'encoder perd menys resolució espacial, el decoder necessita menys capes d'upsampling, i els resultats són més fidedignes amb poques dades d'entrenament.

Per què patches i no volums complets?

Un volum 176×208×176 amb les activacions intermèdies d'un ResNet3D-50 necessitaria desenes de GB de VRAM. Amb patches 32³ l'ús de memòria es redueix dràsticament (~2-4 GB per batch de 4), fent viable l'entrenament en GPUs estàndard.

Per què MedicalNet com a encoder?

A diferència de models com ResNet entrenats amb ImageNet 2D, MedicalNet està pre-entrenat amb volums mèdics 3D (CT/MRI). Les seves convolucions 3D i els seus pesos ja codifiquen patrons anatòmics volumètrics, el que el fa un extractor de features ideal per al nostre domini sense necessitat de reentrenar-lo.

Per què L1 i no L1+Perceptual?

No existeix un VGG3D pre-entrenat equivalent al VGG19 2D. La pèrdua perceptual requereix un model de features entrenat, i en 3D mèdic no n'hi ha de disponibles. L1 és robusta, estable, i produeix bons resultats per SR quan l'objectiu és fidelitat numèrica (PSNR).

Per què sliding window amb overlap?

Sense overlap, les fronteres entre patches adjacents mostren artefactes visibles (discontinuïtats). Amb overlap de 8 vòxels i promitjat, les transicions són suaus i el volum reconstruït és continu.
